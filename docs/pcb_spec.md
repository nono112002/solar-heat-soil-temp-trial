# PCB 設計仕様書 - PicoBox 基板

## 目的
DS18B20 センサーの配線煩雑さを解消するための専用基板。
Pico W + SDカード + 保護回路 + LED + バックアップ電源回路 をひとつの基板にまとめる。

**基板構成**: 1枚統合（センサー・SD分離なし）
- センサー側は GP28 の1信号線のみのため分離不要

---

## 外部接続（フェルール端子 + WAGO/Phoenix）

| 端子 | 信号 | 備考 |
|---|---|---|
| 主電源 5V / GND | 電源入力 | 2ピン端子台 |
| バックアップ電池 + / - | 単3×4本 6V | 2ピン端子台 |
| DS18B20 × 7本 | VCC / DATA / GND | 3ピン端子台 × 7 |

---

## GPIO ピン割り当て

| GPIO | 機能 | 接続先 |
|---|---|---|
| GP2  | SPI0 SCK  | SDカード |
| GP3  | SPI0 MOSI | SDカード |
| GP4  | SPI0 MISO | SDカード |
| GP5  | SPI0 CS   | SDカード |
| GP6  | SD カード検出（挿入時 LOW） | SDモジュール CD ピン + PULL_UP |
| GP15 | TX LED（送信時点滅） | LED + 330Ω → GND |
| GP28 | 1-Wire BUS（全センサー共通） | DS18B20 最大7本 + 4.7kΩ → 3V3 |
| GP_MON（未定） | 主電源監視 | 分圧回路 → GPIO |
| VSYS | 電源入力 | ダイオードOR回路出力から |
| GND  | グラウンド | 共通 |

> **変更履歴**: GP16-19 → GP2-5（2026-03-29）/ GP20 → GP6（SD card detect）

## 回路構成

### 電源入力・バックアップ切り替え（ダイオードOR回路）

```
主電源 5V ─[逆接防止 D1: 1N5819]─[ポリスイッチ 500mA]─┐
                                                        ├─→ 5V ライン
電池   6V（単3×4本）─────────────[D2: 1N5819]───────────┘
```

- 主電源あり: D1通過後 4.7V > D2通過後 5.7V... ※下記参照
- 主電源なし: 自動的にD2側（電池）から供給、マイコン無停止

> **電圧の優先順位**: 電圧の高い方が自動的に勝つ。
> 電池(6V→5.7V) > 主電源(5V→4.7V) なので**電池が常に優先**されてしまう。
> → 主電源は逆接防止のみのダイオードとし、電池側だけにSchottkyを入れる構成を推奨:
>
> ```
> 主電源 5V ─[逆接防止のみ: 1N5819]──────────────────────┐
>                                                         ├─→ 5V ライン
> 電池   6V（単3×4本）──────────[D2: Schottky 1N5819]────┘
> ```
> 主電源5Vが生きていればラインは5V固定（電池のD2は逆バイアスで遮断）
> 主電源が落ちると電池→D2→5.7Vに切り替わる

### 3.3V レギュレーター（外部LDO）

**AMS1117-3.3 → MCP1703-3302 に変更**
理由: AMS1117はドロップアウト1.5V（要最低4.8V入力）だが、
ダイオードOR後の主電源が4.7Vになるため不足。
MCP1703-3302はドロップアウト250mV（4.5V入力でもOK）。

```
5V ライン → MCP1703-3302（SOT-23）→ 3.3V
              入力: 100nF セラコン
              出力: 10µF セラコン
```

この3.3Vを SD カード・DS18B20・LEDに供給（PicoのWiFiとは完全分離）。

### 主電源監視（電源断アラート用）

```
主電源 5V ─[R1: 10kΩ]─┬─[R2: 10kΩ]─ GND
                        └──→ GP_MON（GPIO、約2.5V → HIGH認識）
```

- 主電源あり → GPIO HIGH
- 主電源なし → GPIO LOW → ファームウェアがMQTTアラート送信（電池残量中に送出）

### DS18B20 接続部（GP28 単一バス）

```
3V3（外部LDO出力）→ 4.7kΩ → GP28（データライン）
                              ↓
DS18B20コネクタ × 7個（3ピン端子台: VCC / DATA / GND）
※全本バス共有のため DATA は全て GP28 に接続
```

### TX LED

```
GP15 → 330Ω → LED(赤or青) → GND
```

### SDカードモジュール（物理ピン4〜9 一列接続）

Pico W 物理ピン 4〜9 が SD カードの全信号ピン＋GND と連続している。

| 物理ピン | GPIO | 機能 |
|---|---|---|
| 4 | GP2 | SCK  |
| 5 | GP3 | MOSI |
| 6 | GP4 | MISO |
| 7 | GP5 | CS   |
| 8 | GND | GND（SDモジュールGND） |
| 9 | GP6 | CD（カード検出） |

→ **6ピン一列ソケット1個でSDモジュールの信号線+GND完結**。VCCのみ別途3.3Vラインから引く。

## 電源設計 注意事項（実機確認より）

### 問題: Pico W 内蔵 3V3 レールの電流供給能力不足

**症状**: SD カードマウント失敗 / "timeout waiting for v2 card" / "no SD card"

**原因**:
- Pico W の CYW43 WiFi チップが 3V3 レールを共有
- WiFi 動作中は 3V3 が 2.9V 以下に降下（**実測 2026-03-29: ブレッドボードで 2.9V 確認**）
- SD カードは電源投入時に ~100mA の突入電流が発生

**2026-03-29 実機トラブルシューティング記録**:
- Freenove 拡張ボードの 3V3 端子台の**ねじが緩んでいた**ことが電圧降下の主因のひとつ
- ブレッドボード実装時は接触抵抗が加わり 2.9V まで低下
- ジャンパワイヤ直結 + WiFi OFF シーケンスで解消
- **教訓: ねじ端子台は必ず締め付けトルクを確認すること**

**PCBでの根本対策**:
- SD・センサーの3.3VはMCP1703-3302（外部LDO）から供給 → PicoのWiFi干渉と完全分離
- ファームウェアの`wifi_off()`→SDマウントシーケンスは引き続き維持

## サージ・保護回路

### 入力サージ対策
```
5V IN → TVSダイオード（5.0V, 単方向）→ GND
推奨部品: P6KE6.8A または SMBJ5.0A（SMD）
```

### DS18B20 ライン保護
```
GP28 → 100Ω直列抵抗 → 1-Wire バス
理由: 静電気・誤配線時の GPIO ピン保護
```
