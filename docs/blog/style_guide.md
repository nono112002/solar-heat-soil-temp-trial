# 文章スタイルガイド

野々山昭太のnote記事における文章のニュアンス・語り口調の基準。
1本目の最終稿（2026-04）を正解として抽出。

---

## 基本の文末

**ベースは丁寧体（です・ます）**

```
✅ 「実際に測ったことがなかったです。」
✅ 「なるほどと思う内容が多かったです。」
✅ 「確かめたいところです。」
```

---

## 避ける表現

**「〜なんですよね」は使わない**

UEPON風の語り口として候補に上がるが、このブログには合わない。
読者に対して馴れ馴れしくなりすぎる。

```
❌ 「実際に測ったことがなかったんですよね。」
✅ 「実際に測ったことがなかったです。」

❌ 「だからこそ自分で測る意味があるんですよね。」
✅ 「だからこそ自分で測る意味があると思っています。」
```

---

## 感情・本音の出しかた

**「最後に」や「なぜやるか」など、個人的な部分では文体を崩してよい。**

少し荒削りで、整えすぎていない文章のほうが誠実さが伝わる。

```
✅「自分の畑の、自分の施工条件での数字はどこにもないんで
   ほんとに効いてるのか温度があがってるのかってわからんもんです。」

✅「規模や土質が違えば結果も変わるのかなと思うと、
   自分で測る意味があるんでしょう。」
```

ポイント：
- 「〜ないんで」「わからんもんです」「〜のかなと思うと」など口語を混ぜる
- 文を整えすぎない。少し流れるくらいがちょうどいい
- ただし本文中（説明・設計部分）では使わない。締めや動機の部分だけ

---

## 断言と保留のバランス

**事実には断言、効果・推測には保留**

```
✅「地温が40〜50℃以上になると、病原菌が死滅または抑制されます。」（事実）
✅「発酵熱の上乗せがあれば、温度に何らかの違いが出るかもしれません。」（推測）
✅「違いが出るかどうかも含めて確かめたいところです。」（保留）
```

「かもしれません」「〜とされています」「〜とのことです」を適切に使って
言い切らない誠実さを保つ。

---

## テクニカル用語の扱い

農家・農業系読者を想定。専門用語は使うが、初出時に一言添える。

```
✅「Raspberry Pi Pico W（マイコンボード）に」
✅「DS18B20という防水温度センサーを」
✅「（積算温度は日中最高温度の足し算です）」
```

カッコ書きで補足する形が読みやすい。

---

## リズム・文の長さ

短文と中文を交互に混ぜる。長文は避ける。

```
✅「雑草は減った気がする。土がふかふかになった気もする。
   でも「気がする」止まりで、数字で確かめたことは一度もありませんでした。」
```

段落は短めに。1段落3〜4文が目安。

---

## 見出しの書き方

- 体言止め or 疑問形
- 読み飛ばしOKの案内を入れてよい：「（知ってる方は読み飛ばしてください）」
- 番号付き小見出しは「① ② ③」形式

---

## アドバイス・提案をするとき

**忖度なしで答える。**

読者（野々山）が「これどう思う？」と聞いたとき、遠慮して曖昧にしない。
「やっても意味が薄い」「今は不要」「別の方法がいい」と思ったら、そう言う。

理由を添えて、代替案があれば提示する。最終判断は本人がする。

```
✅「ガーバー単体を公開しても、うれしい人はほぼいないです。理由は〜」
❌「公開するのも一つの選択肢かもしれませんね」
```
