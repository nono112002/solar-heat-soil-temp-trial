# 畑に置ける温度センサーを自作した。回路設計の記録。

---

## 1. 何のために作ったか

太陽熱養生の実験で、土の中の温度を継続的に記録することが目的です。

実験の概要は[前の記事](https://note.com/nono399/n/ne2afba002e9d)に書いています。

メイン機能
3区画それぞれに温度センサーを7本刺して、30分ごとに自動記録します。
WiFiが届かなければSDカードに書き溜めて、復帰後にまとめて送信する。
これを3台作って同時に動かします。

市販の温度ロガーでも記録はできますが、複数区画を同時にクラウドへ飛ばして比較できるものは高価でした。

---

## 2. この回路が何をしているか

回路の中身を部品ごとに説明します。KiCADで描いた回路図のポイントを、できるだけ平易に書きます。

### DS18B20（温度センサー）

防水プローブ付きのデジタル温度センサーです。土に直接刺して使います。マイコンボードのピンに1本ずつ直結する構成にしました。センサー1本につきピン1本を使うシンプルな設計です。

### Raspberry Pi Pico W（マイコンボード）

回路の頭脳にあたる部分です。センサーの値を読み取り、SDカードに書き込み、WiFi経由でクラウドにデータを飛ばします。MicroPythonというプログラミング言語で動かしています。

### MCP1703（LDO：電圧調整IC）

5Vの入力から3.3Vを作る部品です。センサーとSDカード専用の電源として使います。Pico W内蔵のWiFiチップが電源を大きく揺らすため、外部で独立した3.3Vを用意する必要がありました（後述）。

### TVSダイオード（サージ保護）

雷や電源ON/OFFのスパイクから回路を守る部品です。過電圧が来た瞬間にクランプして、後段の回路へのダメージを防ぎます。屋外設置なので省略できない部品です。

### ポリスイッチ（自動復帰ヒューズ）

過電流が流れたとき自動的に電流を遮断し、原因が解消されると自動復帰します。一般的なヒューズと違い、切れたら交換という手間がありません。フィールドに設置するシステムには便利な部品です。

### 分圧回路（主電源監視）

抵抗2本で5V入力を約2.5Vに落として、マイコンのアナログ入力ピン（GP26）で監視します。主電源が落ちて電池に切り替わると、入力電圧が5Vから約4.2Vに下がるため、GP26の電圧も約2.5Vから約2.1Vに低下します。ファームウェアはこの電圧低下をADCで検知して、クラウドにアラートを飛ばします。電池が残っているうちに通知が届く仕組みです。

## 3. KiCADで回路図を描いた

回路が決まったら、KiCADという無償の回路CADで回路図を描き、基板レイアウトに落とし込みました。

KiCADの使い方そのものはここでは書きません。公式サイトからダウンロードできます。
→ [KiCad EDA](https://www.kicad.org/)

回路図が完成したらガーバーファイル（基板製造用データ）を出力して、JLCPCBというサービスに発注しました。5枚で$4.64（送料込み）でした。

## 4. 工夫したポイント

**WiFi用とSDカード・センサー用で電源を分離した**

Pico W内蔵のWiFiチップは動作中に電源電圧を大きく揺らします。SDカードとDS18B20は安定した3.3Vが必要なので、外部LDO（MCP1703）で独立した電源ラインを作りました。

**センサーを1本1本独立したピンに割り当てた**

複数センサーを1本の信号線にまとめる方式だと、センサーごとの固有IDを管理する手間が増えます。今回は7本のセンサーにGP8〜GP14を1対1で割り当てました。どのピンのセンサーが異常かが一目でわかり、ファームウェアもシンプルになります。

**電池バックアップの持ちを計算した**

単3電池3本（4.5V）をバックアップとして積んでいます。平均消費電流から計算すると3〜4日分。毎日畑に行かなくてよいラインを意識して選定しました。

---

## 5. 詰まったポイント

**SDカードがマウントできなかった**

最初、SDカードは何度やっても認識されませんでした。原因は2つ重なっていました。

ひとつはWiFiチップによる電源の揺れ。WiFi動作中に3.3Vが2.9Vまで落ちていることを、テスターで実測してはじめてわかりました。もうひとつは端子台のネジの緩みによる接触不良です。

電圧は実測するまで気づきませんでした。**テスターで測ることを最初からやっておけばよかった**、というのが正直な感想です。

ひとまずソフトウェアで回避しました。WiFiをオフにしてからSDをマウントし、送信のときだけWiFiをオンにする流れにしたところ安定しました。PCBでは電源分離で根本から解決しています。

```
WiFi OFF → SD マウント → センサー読み取り → SD書き込み
→ WiFi ON → クラウド送信 → WiFi OFF → 30分待機
```
