# Pico W + DS18B20 + SDカード + Ambient をつなぎきった日、そしてPCB設計の思考メモ

**2026-04-01 / 太陽熱養生プロジェクト**

---

## やったこと

Raspberry Pi Pico W に DS18B20 温度センサーを7本つなぎ、SDカードに記録しながら Ambient にデータを飛ばす、という一連の流れを動作確認した。
ついでにPCB設計の仕様もほぼ固めた。

結論から言うと、**ファームウェアは動いた。ただし道のりがなかなか険しかった。**

---

## なぜやるか

太陽熱で地面を養生する農業実験で、土中の温度分布を記録したい。
中央・エッジ・深さ方向に合計7本のDS18B20を刺し、30分ごとにSDカードとクラウド（Ambient）の両方に記録する。

電源断でも記録が途切れないこと、現場に行かなくても温度がわかること、この2点がシステムの核。

---

## ハマったポイントと設計の意図

### 1. SDカードがマウントできない問題

最初、SDカードは何度やっても `timeout waiting for v2 card` か `no SD card` で失敗し続けた。

ブレッドボードの配線を疑い、GND接続を確認し、SDカードを変え、ライブラリを入れ直し……それでも失敗。

**原因は2つ重なっていた。**

ひとつは**Pico W の CYW43 WiFi チップが同じ3.3Vレールを共有している**こと。WiFi が動いているとき3.3Vが2.9Vまで落ちていた（テスターで実測）。SDカードの初期化には安定した3.3Vが必要で、これに耐えられなかった。

もうひとつは、もっと単純な話で**拡張ボードの端子台のネジが緩んでいた**。これで接触抵抗が上がり、さらに電圧が落ちていた。

**解決策はシーケンス制御。** `wifi_off()` を呼んでからSDをマウントする。送信のときだけ `wifi_on()` → 送信 → `wifi_off()` という流れにした。WiFiとSDカードが同時に電力を奪い合わない構成にすることで、安定した。

```
WiFi OFF → SD マウント → センサー読み取り → SD書き込み
→ WiFi ON → Ambient送信 → WiFi OFF → 30分待機
```

ソフトウェアで解決したが、PCBでは根本から直す予定（後述）。

---

### 2. センサーIDの管理方法

DS18B20は1-Wireで複数本を同一バスにつなげる。バスをスキャンすると64bitのROMコードが返ってくる。これをどう管理するか。

最初は config.py に直書きしようとしたが、3台のPicoそれぞれでファームウェアを書き換えるたびにIDを確認してコードを書き直す…これはミスが出る。

**SDカードに `sensor_map.json` を置く方法にした。**

```json
{
  "zone": "zone-a",
  "sensors": {
    "286a535200000016": "S1_center_10cm",
    "28983bc0000000c3": "S2_center_25cm",
    ...
  }
}
```

ファームウェアは起動時にSDカードからマッピングを読み込むだけ。Pico3台でファームウェアは共通、SDカードの中身だけ変える。これでヒューマンエラーのリスクが大きく下がった。

---

### 3. ピン配置の設計

GPIO のピン番号を決めるときに気づいたことがある。

Pico W の**物理ピン4〜10が連続している**：

| 物理ピン | GPIO | 用途 |
|---|---|---|
| 4 | GP2 | SD SCK |
| 5 | GP3 | SD MOSI |
| 6 | GP4 | SD MISO |
| 7 | GP5 | SD CS |
| 8 | GND | GND |
| 9 | GP6 | SD カード検出 |
| 10 | GP7 | DS18B20 1-Wire |

SDカードの4本SPI信号＋GND＋カード検出＋センサーバスが**7ピン連続で全部片側に集まる**。

PCBを作るとき、この7ピンをひとつのコネクタでまとめて引き出せる。配線が整理されて、作業ミスも減る。「なんとなくピンを決める」のではなく、**物理的なレイアウトから逆算してピンを決める**のが今回の設計のポイントだった。

---

## PCB設計の方針（次フェーズ）

動作確認が取れたのでPCBに入る。構成を整理すると：

**上流電源ボックス（1台）**
- 24V → 5V DC-DC変換
- 電池バックアップ（単3×3本）をショットキーダイオードOR回路で並列接続
- 主電源が落ちた瞬間に電池に切り替わる（マイコン停止なし）
- 3台のPicoBoxに5Vバスで給電

**PicoBox（3台、同一設計）**
- Pico W ソケット実装
- MCP1703-3302（LDO）で5V→3.3V生成（ドロップアウト250mV）
- SDカード・センサーはこの独立3.3Vから給電（WiFi干渉を根本から断つ）
- 全端子をWAGO/Phoenix端子台＋フェルール端子

**AMS1117ではなくMCP1703を選んだ理由：**
ダイオードOR回路を通すと主電源5Vが4.7Vに落ちる。AMS1117はドロップアウトが1.5Vのため最低4.8V必要で、4.7Vでは動作が保証できない。MCP1703はドロップアウト250mVなのでギリギリではなく余裕を持って動く。部品1個の選定に理由があることを記録しておく。

## 結果・気づき

- ファームウェアは完成、30秒ごとにSD書き込みとAmbient送信が安定動作
- 電圧問題はソフトウェアで一時回避、PCBで根本解決予定
- 「ブレッドボードのネジが緩んでいた」という物理的な原因が意外と大きかった。**電圧をテスターで実測する習慣は大事**
- ピン番号は連続性を意識して選ぶと配線がきれいになる
